User Stores the synced central-auth identity plus portal-specific attributes like business group, IOU, and account.

SkillMaintainer Represents ownership assignments, including who assigned the maintainer for a given skill.

Approved catalog `agents_catalog` is the live directory surface. Only approved skills land here after the workflow completes.
Pending + reviewed history `agents_pending` and `agents_reviewed` track approval state, review timestamps, rejection reasons, and analysis metadata.
Record granularity Redis stores whole JSON records keyed by skill ID, which keeps the review workflow fast and simple to read from the app layer.
